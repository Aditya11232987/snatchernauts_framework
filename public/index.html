Ren’Py · Shader-driven VN/Adventure Toolkit

Neo-noir tools for interactive stories

Snatchernauts is a focused framework for building modern, **Snatcher/Policenauts-inspired** visual novels: shader pipeline (CRT, grain, grading, lighting), an interaction/room system, dialogue & cutscenes, and a small API surface to script gameplay.

Browse Repository Report a Bug Documentation View Screens

Shader Pipeline

Post-FX chain (CRT → Grain → Grade → Letterbox) on the gameplay layer; object-level FX (blur, lighting) on a dedicated layer. All passes can animate and compose.

  • Multi-pass GL2 shadersRen’Py 8.4.1
  • Screen & per-object blur
  • Dynamic lighting spots & masks

Interaction & Rooms

Clickable objects, hover prompts, and contextual menus. Cutscene and dialogue systems integrate cleanly with interactions.

  • Room graph + hotspots
  • Interaction dispatcher
  • Cutscene cues & camera FX

API & Extensibility

Small Python API for room/object registration, shader toggles, and event hooks. Designed to be readable and mod-friendly.

  • `register_object(...)`
  • `enable_shader('letterbox', args)`
  • `on_interact(room,obj,action)`

Quick Start

  1. Clone the repo and run with Ren’Py 8.4.1 (GL2 enabled).
  2. Open script.rpy and ui/screens_room.rpy to explore interactions.
  3. Tweak shader presets in shaders/*.glsl and enable via the API.
  4. Replace the demo room art and add hotspots in rooms/.
